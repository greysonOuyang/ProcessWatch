# Code of Conduct

ProcessWatch contributors are expected to communicate professionally, assume good intent, and focus criticism on code and behavior rather than people.

Unacceptable behavior includes harassment, discrimination, threats, deliberate disclosure of private information, publishing credentials or sensitive process data, and repeatedly disrupting technical discussion.

Maintainers may edit or remove content, close discussions, reject contributions, or restrict participation when necessary to protect the project and its contributors. Reports should be made privately to repository maintainers; security issues must follow `SECURITY.md`.

This policy applies to project repositories, issue trackers, review discussions, release channels, and community spaces operated by the project.
